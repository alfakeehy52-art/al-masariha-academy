## الإجابة المختصرة

| السؤال | الجواب |
|--------|--------|
| **هل الموقع مكتمل للإطلاق؟** | **نعم — كمرحلة أولى (Soft Launch)** للعمليات الأساسية |

| **هل أطلقه الآن والباقي لاحقاً؟** | **نعم — موصى به** بشرط قائمة تحقق قبل الإعلان + خطة تحديثات واضحة |

## ما يعمل اليوم (أساس الإطلاق)

| المجال | الحالة | ملاحظة |
|--------|--------|--------|
| الموقع العام | ✅ | رئيسية، فرق، لاعبون، مدربون، أخبار، إعلام، متجر، تواصل |
| انضمام / طلبات | ✅ | نموذج + متابعة + استكمال مرفقات (متقدم) |
| لوحة الإدارة | ✅ | طلبات، لاعبون، كوادر، متجر، إعدادات أكاديمية |
| Auth + RLS | ✅ | نُفّذت سكربتات SQL حسب `RLS_DEPLOY_ORDER.md` |
| النشر | ✅ | `git push` → Cloudflare Pages |
| إعدادات الموقع E1 | ✅ | شعار، نصوص، SEO جزئي |

**الخلاصة:** المنصة **قابلة للاستخدام الفعلي** كأكاديمية على الويب — ليست نموذجاً تجريبياً فارغاً.

## ما لا يمنع الإطلاق لكن يجب معرفته

هذه **فجوات مقبولة لمرحلة 1** إذا وُثّقت وخطّط لها:

| الفجوة | التأثير على الإطلاق | متى تُعالج |
|--------|---------------------|------------|
| روابط السوشيال غير ظاهرة في الموقع | متوسط — الزائر لا يرى قنواتكم | P0 — أيام |
| مراجعة مرفقات ولي أمر/داعم/متطوع ناقصة في الإدارة | **عالٍ إن تستقبل هؤلاء الآن** | P0 قبل استقبالهم |
| وميض الشعار | منخفض — مظهر فقط | P1 |
| صلاحيات = إخفاء قائمة فقط (لا RLS كامل) | متوسط — للفريق الداخلي الصغير مقبول | P2 |
| لا نسخ احتياطي من اللوحة | منخفض — اعتمد Supabase backup | P2 |
| جوال الإدارة ضعيف | متوسط — الإدارة من الجوال أصعب | P1 |
| نشر تلقائي للسوشيال / نشرة | لا يؤثر على الإطلاق | P3 |

**التفاصيل الكاملة:** [`OWNER_FEEDBACK_BACKLOG.md`](OWNER_FEEDBACK_BACKLOG.md)

### موصى به قبل الإطلاق (أسبوع)

- [ ] إظهار واتساب + إنستغرام + تيك توك/سناب في الفوتر

- [ ] اختبار جوال: الصفحة الرئيسية + انضمام + متجر

### يمكن بعد الإطلاق

- [ ] إصلاح وميض الشعار
- [ ] تحسين القائمة الجانبية على الجوال
- [ ] RLS صلاحيات متقدم (R3)
- [ ] نسخ احتياطي من اللوحة
- [ ] تكامل سوشيال / نشرة

## نموذج الإطلاق المرحلي (موصى به)

```text
المرحلة 1 — إطلاق الآن (4–8 أسابيع)
├── الموقع العام + المتجر + الأخبار
├── استقبال طلبات (لاعب / مدرب أولاً)
├── فريق إدارة صغير (مدير + 1–2 موظف)
└── تحديثات أسبوعية من OWNER_FEEDBACK_BACKLOG (P0 ثم P1)

المرحلة 2 — بعد الاستقرار
├── جوال إدارة + PWA
├── صلاحيات RLS كاملة
├── حذف نهائي اختياري + نسخ احتياطي

المرحلة 3 — نمو
├── نشر محتوى متعدد المنصات
├── نشرة بريدية + WhatsApp Business (إن لزم)
└── تطبيق متجر (Capacitor / Flutter WebView)
```

## مقارنة: «مكتمل» vs «جاهز للإطلاق»

| | مكتمل 100% | جاهز مرحلة 1 |
|---|------------|----------------|
| كل الميزات المستقبلية | ✅ | ❌ |
| العمليات الأساسية تعمل | ✅ | ✅ |
| لا أخطاء حرجة معروفة | ✅ | ✅ (مع فجوات موثّقة) |
| UX مثالي على كل الأجهزة | ✅ | ❌ — يتحسن تدريجياً |
| تكامل سوشيال API | ✅ | ❌ — لاحقاً |

**رأي تقني:** لا تؤجل الإطلاق حتى «الكمال» — المشاريع الحية تُطلق عند **قيمة + استقرار أساسي**، وتُحسَّن بالتغذية الراجعة.

## مخاطر إن أطلقت دون P0

| السيناريو | الخطر |
|-----------|-------|
| تستقبل داعم/ولي أمر/متطوع اليوم | الإدارة قد لا تراجع المرفقات الصحيحة |
| فريق كبير بصلاحيات مختلفة | قد يفتح صفحات لا ينبغي (حتى R3) |
| لا نسخ Supabase | فقدان بيانات نادر لكن كارثي — فعّل backup في Supabase |
