# نظام التواصل — تشغيل إداري (ليس «شات» عام)

**آخر تحديث:** 2026-05-22  
**الحالة:** **MVP واجهة منفّذ** — SQL + Drawer إدارة + متابعة طلب.  
**لا يُدمج** مع `contact_messages`.

**SQL مرجعي:** [`CHAT_MVP_SCHEMA.sql`](CHAT_MVP_SCHEMA.sql)

---

## الرؤية

نظام **تشغيل إداري**: تواصل مربوط بسجل رسمي، صلاحيات، audit — ليس واتساباً ولا سوشال ميديا.  
واجهة البداية: **زر «تواصل» → Panel / Drawer** (وليس تطبيق شات كامل من اليوم الأول).

---

## قرارات مقفلة (المالك — 2026-05-22)

### 1) الاحتفاظ بالرسائل — `room_retention_type`

| القيمة | الاستخدام |
|--------|-----------|
| `permanent` | غرف الطلبات والتسجيلات، عقود، قضايا إدارية — **دائم أو طويل جداً** |
| `90_days` | دعم / دعوات مؤقتة — ثم **أرشفة** (إخفاء من القائمة النشطة، الاحتفاظ بالسجل حسب السياسة) |
| `temporary` | غرف قصيرة جداً (اجتماع عاجل) — انتهاء تلقائي |

لا يُجعل كل شيء دائماً لتجنب تضخم DB بلا فائدة.

### 2) دخول ولي الأمر

| المرحلة | الآلية |
|---------|--------|
| **أثناء الطلب** | `request_status` + **PIN** + **رابط** — **بدون حساب** |
| **بعد الاعتماد → لاعب رسمي** | إنشاء **حساب ولي أمر** حقيقي |

### 3) المرفقات (MVP)

| النوع | الحد |
|--------|------|
| صور | 500 KB – 1 MB |
| PDF | 2 – 5 MB |

**ممنوع:** فيديو، ZIP، ملفات تنفيذية (.exe …).

### 4) الأداء — Realtime

| المكان | الآلية |
|--------|--------|
| قائمة الغرف | **Polling خفيف** |
| داخل الغرفة المفتوحة فقط | **Supabase Realtime** |

### 5) حقول الغرفة (من اليوم الأول)

| الحقل | الغرض |
|--------|--------|
| `room_type` | `join_request` \| `guardian` \| `staff` \| `temporary_invite` \| `admin_support` |
| `related_entity_table` | مثلاً `join_requests`, `players`, `guardians` |
| `related_entity_id` | معرّف السجل (UUID أو نص حسب الجدول) |
| `room_retention_type` | `permanent` \| `90_days` \| `temporary` |
| `title_ar` | عنوان عربي للواجهة |

### 6) واجهة MVP (ليست صفحة شات كاملة)

| الموقع | السلوك |
|--------|--------|
| `admin_requests.html` | زر **تواصل** → Drawer محادثة الطلب |
| `player_view_dashboard.html` | زر **تواصل** → Drawer (بعد اعتماد اللاعب) |
| `communications_dashboard.html` | قائمة كل الغرف + Badge (للإدارة) |

## مبادئ ثابتة

1. كل غرفة مربوطة بـ `related_entity_table` + `related_entity_id`.  
2. `chat_audit_log` لكل إجراء حساس.  
3. RLS + أدوار في `chat_members`.  
4. لا سوشال (لا حالات، لا «من متصل»، لا غرف عامة).  
5. فصل تام عن `contact_messages`.  
6. MVP: إدارة ↔ طلب REQ، إدارة ↔ ولي (رابط/PIN)، دعوة مؤقتة — **لا** جماعيات فئات/كادر في v1.

### قبل العرض التشغيلي (مخطط — لم يُنفَّذ بعد)

| البند | التفاصيل |
|--------|----------|
| **Realtime `chat_notifications`** | تفعيل Replication في Supabase عند ربط عداد «غير مقروء» في الشريط الجانبي أو مركز إشعارات |
| **Realtime `chat_messages`** | ✅ مفعّل — يكفي لـ Drawer الحالي |
| **جدول `chat_notifications`** | يُملأ من التطبيق لاحقاً عند إرسال رسالة؛ حالياً الدرج يعتمد على `chat_messages` + polling احتياطي |
