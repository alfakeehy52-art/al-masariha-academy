# نظام التواصل والدردشة — وثيقة مناقشة (غير منفّذ)

**التاريخ:** 2026-05-22  
**الحالة:** للتخطيط بعد اعتماد العرض التشغيلي — **مشروع منفصل**.

## هدف المالك

نظام تواصل احترافي داخل موقع الأكاديمية (ليس شاتاً عاماً مفتوحاً)، مع:

- تقسيم حسب الصلاحيات والفئات
- غرف خاصة لكل فئة (براعم / ناشئين / شباب)
- غرف للكادر (مدربين / إداريين / سائقين / مشرفين)
- إدارة ↔ أولياء الأمور
- غرف مؤقتة أو دائمة لأشخاص خارج النظام (كود أو رابط دخول)
- صلاحيات إدارة: حذف رسائل، كتم، إغلاق غرف، تثبيت، إشعارات
- واجهة حديثة، حالة اتصال، ملفات وصور، أداء جيد

## مقترح مراحل (للمناقشة)

| مرحلة | النطاق | تقنية محتملة |
|--------|--------|----------------|
| **أ** | رسائل إدارية → ولي أمر / لاعب (بريد + إشعار داخل اللوحة) | جداول `messages` + Realtime Supabase |
| **ب** | غرف فئات ثابتة (براعم، ناشئين، …) للكادر المعتمد فقط | عضوية غرفة + RLS |
| **ج** | غرف كادر حسب `staff_type` | ربط `academy_staff` |
| **د** | غرفة دعوة مؤقتة (رابط + PIN + انتهاء) | `chat_invite_tokens` |
| **هـ** | إشعارات push / بريد / واتساب لاحقاً | Queue أو خدمة خارجية |

## مبادئ تصميم

1. **لا دردشة عامة** — كل محادثة في غرفة معرّفة ومراقَبة.
2. **الإدارة مركزية** — إنشاء/إغلاق الغرف، مراقبة المحتوى، سجل تدقيق.
3. **RLS صارم** — المستخدم يرى غرفته فقط؛ الضيف برابط مؤقت scoped لغرفة واحدة.
4. **فصل عن `join_requests`** — التواصل بعد الاعتماد أو لحل طلب محدد.

## أسئلة للمالك قبل التقدير

1. هل الدردشة للجميع أم للمعتمدين فقط؟
2. هل أولياء الأمور يراسلون إدارة واحدة أم مدرب الفئة؟
3. هل اللاعبون القاصرون يشاركون أم الولي فقط؟
4. مدة الاحتفاظ بالرسائل (٣٠ / ٩٠ / سنة)؟
5. هل الملفات الصوتية/الفيديو مطلوبة في الإصدار الأول؟

## تقدير جهد تقريبي

| الإصدار | المدة |
|---------|-------|
| MVP (إدارة ↔ ولي أمر + غرفة فئة واحدة) | 2–3 أسابيع |
| كامل حسب الوصف أعلاه | 6–10 أسابيع |
