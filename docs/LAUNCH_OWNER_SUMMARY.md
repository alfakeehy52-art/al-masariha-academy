# 2) الاشتراكات — قبل الإطلاق vs بعد الاعتماد

### الآن (عرض تشغيلي) — **لا شراء**

| البند | البديل المجاني |
|-------|----------------|
| دومين | `al-masariha-academy.pages.dev` |
| بريد رسمي | Gmail الحالي في التواصل مؤقتاً |
| واتساب | رقم عادي في صفحة التواصل (اختياري) |

| # | البند | تكلفة تقريبية |

| 3 | **واتساب Business** + رقم | شريحة ~30–50 ر.س/شهر |

### دائماً (موجود)

| # | البند | ضروري؟ | تكلفة تقريبية |
|---|-------|--------|----------------|
| 4 | Cloudflare Pages | ✅ موجود | مجاني |
| 5 | Supabase | ✅ موجود (Free كافٍ للبداية) | مجاني |

| 7 | Google Search Console | موصى | مجاني |
| 8 | حسابات سوشيال (إنشاء روابط) | موصى | مجاني |

# 3) ما يحتاجه الموقع قبل الإطلاق

| المجال | المطلوب |
|--------|---------|
| **تقني P0** | صلاحيات · مرفقات · **جوال** · سوشيال «قريباً» · تنظيف تجريبي |
| **تشغيل (بعد الاعتماد)** | دومين · `siteUrl` · Supabase Auth URLs |
| **محتوى** | شعار نهائي · نصوص حقيقية · فرق/مدربون · سياسة خصوصية |
| **اختبار** | مسار: انضمام → موافقة → مرفقات → اعتماد (على الإنتاج) |
| **تنظيف** | حذف بيانات تجريبية (بند 6) |
| **واجهة** | إزالة إشارات «قاعدة البيانات» للزائر (بند 4) |

| بعد الاعتماد | دومين · بريد · سوشيال فعلي · RLS متقدم · نسخ لوحة · تطبيق جوال |
| بعد العرض | وميض شعار · RLS R3 |

# 4) تنظيف الموقع (نظيف وجاهز)

| ما يُزال/يُستبدل | أين |
|------------------|-----|
| نصوص «قاعدة البيانات» / «Supabase» للمستخدم | لوحات الإدارة (تحميل، أخطاء، حذف) → «النظام» / «السجلات» |
| `<code>supabase-config.js</code>` و `docs/*.sql` في الواجهة | `admin_permissions_dashboard.html` → نص إداري بسيط |
| `placeholder="admin@example.com"` | `admin_login.html` → بريدك الرسمي |
| رسائل خطأ تقنية في Toast | استبدال برسائل عربية عامة |
| مجلد `docs/` على الإنتاج | التأكد أنه غير مرتبط من الموقع (يبقى في Git للفريق فقط) |
| `console.error("Supabase...")` | يبقى للمطور — لا يظهر للزائر |

**لا يُحذف من الكود:** `supabase-config.js` وملفات JS — ضرورية للتشغيل، مخفية عن الزائر.

# 6) حذف المدخلات التجريبية — SQL

> **تحذير:** نسخة احتياطية أولاً. نفّذ في Supabase → SQL Editor.  
> **لا يحذف:** `academy_settings` · مستخدمي Auth (احذفهم من Authentication يدوياً).

```sql
-- ═══════════════════════════════════════════════════════════
-- تنظيف بيانات تجريبية — أكاديمية المسارحة
-- راجع الأعداد قبل التنفيذ: SELECT count(*) FROM ...
-- ═══════════════════════════════════════════════════════════

BEGIN;

-- 1) روابط وجداول تابعة
DELETE FROM player_guardians;
DELETE FROM request_completions;
DELETE FROM admin_notifications;

-- 2) طلبات الانضمام
DELETE FROM join_requests;

-- 3) كيانات التشغيل
DELETE FROM players;
DELETE FROM coaches;
DELETE FROM guardians;
DELETE FROM supporters;
DELETE FROM volunteers;

-- 4) كادر (احتفظ بسجلك إن لزم — عدّل WHERE)
-- DELETE FROM academy_staff WHERE email NOT IN ('admin@yourdomain.com');

-- 5) عضوية / مجتمع / فرق / محتوى / متجر
DELETE FROM academy_members;
DELETE FROM teams;
DELETE FROM matches;
DELETE FROM academy_news;
DELETE FROM academy_media;
DELETE FROM contact_messages;
DELETE FROM store_orders;
DELETE FROM store_products;

COMMIT;

-- 6) التخزين: Storage → bucket academy_files → احذف الملفات التجريبية يدوياً
-- 7) Auth: Dashboard → Authentication → Users → احذف حسابات التجريب
```

### بعد SQL

| الخطوة | أين |
|--------|-----|
| حذف ملفات مرفوعة | Storage → `academy_files` |
| حذف مستخدمي تجريب | Authentication → Users |
| الإبقاء على إعدادات الموقع | جدول `academy_settings` |
| تحديث `adminEmails` | `supabase-config.js` → بريد المدير فقط |

# 7) ضبط الجوال — P0 للعرض التشغيلي

> **المشكلة الحالية:** على الجوال التجربة «مزعجة» — تمرير أفقي، قوائم، جداول عريضة، لوحة إدارة صعبة.

### أولويات الإصلاح (بالترتيب)

| # | المنطقة | المطلوب | الملفات المرجعية |
|---|---------|---------|------------------|
| 1 | **القائمة الجانبية — إدارة** | إغلاق بالنقر خارج الدرج + Escape؛ عدم الوميض | `admin_sidebar.js` |
| 2 | **جداول لوحة الإدارة** | على عرض &lt;768px: بطاقات بدل جداول عريضة (min-width 900–1100px) | صفحات admin + `admin_theme.css` |
| 3 | **الموقع العام** | مراجعة: رئيسية، انضمام، استكمال مرفقات، متجر، تواصل | `shared-layout.css`, `request_completion.html`, `al_masariha_join_page.html` |
| 4 | **أزرار ولمس** | ارتفاع لمس ≥44px؛ مسافات؛ عدم تكبير أفقي للصفحة | CSS عام |
| 5 | **تسجيل دخول إدارة** | حقول وزر مريحان على شاشة ضيقة | `admin_login.html` |
| 6 | **اختبار أجهزة** | iPhone/Android: عرض + إدارة + رفع مرفق من الجوال | يدوي قبل الاجتماع |

### ما يُؤجَّل (مرحلة تطبيق)

- PWA + `manifest.json`
- تطبيق Apple/Google Store
- تصميم «كأنه تطبيق» كامل (شريط سفلي ثابت…)

### معيار «جاهز للعرض على الجوال»

- [ ] لا تمرير أفقي مزعج في الصفحات الرئيسية
- [ ] ☰ القائمة الجانبية تفتح وتُغلق بسلاسة
- [ ] إرسال طلب انضمام + رفع مرفق يعمل من الجوال
- [ ] مراجعة طلب واحد في لوحة الإدارة ممكنة من الجوال
