# المرحلة G — بيانات تشغيلية

**قاعدة:** لا حذف بيانات بدون **موافقة صريحة** من المالك + نسخ احتياطي أولاً.

---

## 1) نسخ احتياطي (إلزامي قبل أي تنظيف)

### في Supabase

1. **Dashboard → Database → Backups** — تأكد أن النسخ الاحتياطي التلقائي مفعّل (خطة Pro).
2. **Database → Backups → Download** أو استخدم نقطة استعادة عند الحاجة.

### تصدير يدوي (موصى به قبل التنظيف)

1. **Table Editor** — للجداول الحساسة: `join_requests`, `players`, `request_completions`, `academy_settings`.
2. أو **SQL** → تصدير نتائج `SELECT *` للمعاينة والأرشف.

### Storage

- **Storage → `academy_files`** — حمّل مجلدات تجريبية مهمة إن وُجدت عقود حقيقية مختلطة.

---

## 2) تنظيف تجريبي (بموافقة فقط)

**الملف:** `docs/LAUNCH_CLEANUP_TEST_DATA.sql`

| قبل التنفيذ | |
|-------------|---|
| ⬜ موافقة المالك مكتوبة («أوافق على حذف التجريبي») | |
| ⬜ نسخة احتياطية مأخوذة | |
| ⬜ شغّلت SELECT المعاينة وراجعت الأعداد | |
| ⬜ عدّلت سطر `academy_staff` إن أردت الإبقاء على بريدك | |

**بعد SQL:**

- Auth → Users → احذف حسابات تجريبية
- Storage → احذف ملفات تجريبية
- `supabase-config.js` → `adminEmails` للمدير فقط
