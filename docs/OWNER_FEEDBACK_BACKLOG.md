## البنود التفصيلية

### 1) وميض الشعار عند تحديث الصفحة

| | |
|---|---|
| **الوصف** | عند F5 يظهر الشعار الافتراضي ثم يتبدل للشعار المرفوع من اللوحة |
| **السبب** | HTML يبدأ بـ `academy-logo.svg` ثم `js/academy-settings.js` يجلب `logo_url` من Supabase |
| **الحل المقترح** | إخفاء الشعار حتى التحميل، أو `localStorage` لآخر شعار + تحديث من السيرفر |
| **الملفات** | `shared-layout.js`, `js/academy-settings.js`, `admin_sidebar.js` |
| **الحالة** | ⬜ لم يُنفَّذ |
| **الأولوية** | P1 |

---

### 2) تيك توك + سناب شات + بقية التواصل الاجتماعي

| | |
|---|---|
| **الوصف** | إضافة تيك توك وسناب مع إنستغرام وX وواتساب وربطها بالموقع |
| **الوضع الحالي** | الإعدادات: `whatsapp_url`, `instagram_url`, `twitter_handle` فقط — **ولا تظهر في الفوتر/التواصل** |
| **الحل المقترح** | ① إظهار الروابط الحالية في `shared-layout.js` + صفحة التواصل ② حقول SQL جديدة: `tiktok_url`, `snapchat_url`, `youtube_url` ③ لوحة الإعدادات + `academy-settings.js` |
| **ملاحظة** | لا حاجة لـ «صفحة تيك توك» داخل الموقع — روابط رسمية كافية |
| **الحالة** | ⬜ لم يُنفَّذ |
| **الأولوية** | P0 |

---

### 3) القائمة الجانبية (إظهار/إخفاء + جوال)

| | |
|---|---|
| **الوصف** | سلوك غير مناسب: تظهر وتختفي (تمرير الماوس / جوال) |
| **الوضع الحالي** | `admin_sidebar.js`: ثابتة على الديسكتوب؛ على الجوال درج بـ ☰ — **لا hover collapse**؛ لا إغلاق بالنقر خارج الدرج |
| **الحل المقترح** | ديسكتوب: ثابتة أو طي يدوي (زر) — ليس hover. جوال: إغلاق overlay + Escape + بعد الانتقال |
| **الملفات** | `admin_sidebar.js`, `admin_theme.css` |
| **الحالة** | ⬜ لم يُنفَّذ |
| **الأولوية** | P1 |

---

### 4) تجربة جوال كاملة (كأنها تطبيق — ثم تطبيق متجر لاحقاً)

| | |
|---|---|
| **الوصف** | ضبط الموقع بالكامل للجوال؛ لاحقاً PWA / تطبيق |
| **الوضع الحالي** | الموقع العام جيد نسبياً (`shared-layout.css` 980px). الإدارة: جداول عريضة، تمرير أفقي |
| **الحل المقترح** | **مرحلة أ:** جداول→بطاقات، أزرار لمس 44px، توحيد breakpoints. **مرحلة ب:** `manifest.json` + Service Worker + Capacitor |
| **الحالة** | ⬜ جزئي |
| **الأولوية** | P1 (أ) · P3 (ب تطبيق) |

---

### 5) صلاحيات الموظف الحالي — «لا توجد صفحة»

| | |
|---|---|
| **الوصف** | كيف أضيف صلاحية لموظف؟ لا صفحة إعدادات صلاحيات وحذف |
| **الوضع الحالي** | **الصفحة موجودة:** `admin_permissions_dashboard.html` (للمدير العام فقط). R1+R2-lite: إخفاء القائمة حسب `academy_staff.role` |
| **خطوات التشغيل** | ① `docs/STAFF_PANEL_ROLES.sql` في Supabase ② بريد في `adminEmails` (`supabase-config.js`) ③ لوحة الصلاحيات → اختيار الدور ④ خروج/دخول |
| **ما ينقص** | R3: RLS حسب الدور، منع فتح URL مباشرة، حذف سجلات صلاحيات |
| **المرجع** | `docs/RBAC_PLAN.md`, `js/panel-access.js`, `js/admin-permissions.js` |
| **الحالة** | ✅ صفحة موجودة · ⬜ R3 · ⬜ توثيق واضح للمالك |
| **الأولوية** | P0 (تشغيل) · P2 (R3) |

---

### 6) زر حذف للكيانات (لاعب، كادر، داعم…)

| | |
|---|---|
| **الوصف** | لا يوجد حذف عند الرغبة |
| **الوضع الحالي** | **حذف نهائي:** لاعب، مدرب، فريق، أخبار، مباريات، متجر. **تعطيل فقط:** ولي أمر، داعم، متطوع، كادر (`entity_management_app.js`) |
| **الحل المقترح** | الإبقاء على التعطيل كافتراضي + «حذف نهائي» للمدير العام فقط + تأكيد مزدوج + RLS |
| **الحالة** | ⬜ حذف نهائي للكيانات الناعمة |
| **الأولوية** | P2 |

### 9) نشر محتوى من الموقع إلى السوشيال + نشرة + واتساب

| | |
|---|---|
| **الوصف** | مثلاً مباراة → بث تيك توك، صورة إنستغرام، مقال X — التحكم من لوحة واحدة |
